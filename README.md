## 🤖 AI Integration

<div align="center">

**OpenAI GPT-4o** powers all 6 AI engines inside FlowPilot AI.

</div>

| # | Engine | Role | How GPT-4o Is Used |
|---|---|---|---|
| 1️⃣ | **Payment Prediction** | Forecasts payment arrival, flags risk | Analyzes behavioral patterns + outputs risk JSON |
| 2️⃣ | **Anomaly Detection** | Flags unusual expenses | Compares to baseline, classifies reason |
| 3️⃣ | **AI Recovery** | Drafts collection messages | Tone-calibrated prompts × 3 stages × 3 languages |
| 4️⃣ | **Trust Scoring** | Live 0–100 client scores | Weights timing, trend, consistency, disputes |
| 5️⃣ | **AI CFO Advisor** | Daily briefs + recommendations | Aggregated snapshot → structured JSON output |
| 6️⃣ | **Expense Intelligence** | Classifies spend + detects leakage | Category inference + anomaly reasoning |

**Cost optimization strategy:**
- All responses cached by `(user_id, input_hash)` in Supabase
- 8 AM daily brief pre-generated → subsequent views return cached at **<100ms**
- Recovery messages cached per `(invoiceId, stage, language)` for 30 minutes
- Streaming animation gives live feel even on cached responses

## 🚀 Getting Started

### Prerequisites

```bash
node --version    # v20.0.0 or higher required
npm --version     # v9.0.0 or higher required
```

You need accounts at:
- [Supabase](https://supabase.com) — free tier is enough
- [OpenAI](https://platform.openai.com) — API key (GPT-4o access)
- [Vercel](https://vercel.com) — for deployment
- [Resend](https://resend.com) — for email notifications

### Step 1 — Clone the repo

```bash
git clone https://github.com/yourusername/flowpilotai.git
cd flowpilotai
```

### Step 2 — Install dependencies

```bash
npm install
```

### Step 3 — Set up environment variables

```bash
cp .env.example .env.local
# Open .env.local and fill in your values
```

### Step 4 — Set up Supabase database

1. Create a new Supabase project at [supabase.com](https://supabase.com) — select **Singapore** region
2. Go to **SQL Editor** in your dashboard
3. Run the migrations in order:

```bash
# Run schema (copy from supabase/migrations/001_schema.sql)
# Run RLS policies (copy from supabase/migrations/002_rls.sql)
```

### Step 5 — Configure Supabase Auth

```
Dashboard → Authentication → Providers → Email
→ Enable ✓
→ Confirm email: OFF (for demo speed)

Dashboard → Authentication → URL Configuration
→ Site URL: http://localhost:3000
```

### Step 6 — Start development server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) ✅

### Step 7 — Seed demo data

Sign up → click "Skip to Dashboard" on onboarding → the seed route runs automatically.

Or manually:

```bash
curl -X POST http://localhost:3000/api/seed \
  -H "Cookie: your-session-cookie"
```

This creates 3 clients · 8 invoices · 84 transactions · 4 suppliers · 12 automation rules · pre-warmed AI cache.

### ⚡ Core Recovery Loop (23 steps, ~3.5 seconds end-to-end)

```
1. Owner opens Overdue Radar → sees Nexus, 11 days late, LKR 185,000
2. Clicks "Recover" → Recovery Center loads in 200ms (cached)
3. Selects "සිංහල" → GPT-4o translates Stage 2 message
4. Taps "Open in WhatsApp" → wa.me deeplink fires
5. Client receives message, clicks card payment link
6. Client opens /pay/[invoiceId] (public, no login)
7. Taps "Pay by Card" → MPGS creates Mastercard session
8. Client enters card on Mastercard's secure hosted page
9. Mastercard redirects → server verifies order status
10. Invoice → PAID · Supabase Realtime pushes to browser
11. 🎉 Confetti · Toast: "LKR 185,000 received from Nexus Traders"
```

## 🔒 Security

| Layer | Implementation |
|---|---|
| **Authentication** | Supabase Auth · HTTP-only cookies · No localStorage tokens |
| **Authorization** | PostgreSQL Row Level Security on all 12 tables |
| **Multi-tenancy** | Every query filtered by `auth.uid() = user_id` at database level |
| **Secret management** | All API keys server-side only · Zero `NEXT_PUBLIC_` for secrets |
| **Input validation** | Zod schema on every API route before any database access |
| **AI output validation** | Zod parsing on every GPT-4o JSON response before persistence |
| **Card data** | Never stored by FlowPilot · Direct to Mastercard PCI-DSS infrastructure |
| **SQL injection** | Structurally impossible via Supabase parameterized queries |
